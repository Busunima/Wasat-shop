## Миниатюры изображений товаров (FR-A02)

Авто-генерация миниатюр — расширение **Storage Resize Images** (конфиг
`firebase/extensions/resize-product-images.env`, деплой:
`firebase deploy --only extensions`). Соглашение имён: рядом с оригиналом создаются
`<имя>_200x200.webp` и `<имя>_800x800.webp` (каталог/корзина используют 200x200,
карточка — 800x800). Клиент строит URL миниатюры по соглашению
(`core/util/ImageUrls.kt`) и откатывается на оригинал, если миниатюры ещё нет
(расширение не развёрнуто или ещё обрабатывает файл).

## Источник MAU — users.lastActiveAt (FR-S04)

Глобальная аналитика суперадмина требует MAU, но в данных нет следа активности.
Решение: `requireAuth` после успешной проверки ID-токена делает best-effort запись
`users/{uid}.lastActiveAt = serverTimestamp()` (`services/activity.touchUser`,
без await, не бросает — не влияет на ответ). MAU = число пользователей с
`lastActiveAt` за последние 30 дней (count-агрегат). На масштабе MVP допустима запись
на каждый авторизованный запрос; при росте — троттлинг (писать не чаще раза в сутки на
uid) или отдельный дневной агрегат активных пользователей. GMV платформы суммирует
`revenue` дневных агрегатов всех магазинов в минорных единицах — на MVP без приведения
валют (магазины могут быть в разных валютах); приведение по курсу — позже.

## AI-ассист контента — Claude API, env-gated (FR-A12)

Генерация описаний товара — официальный SDK `@anthropic-ai/sdk`, модель
`claude-opus-4-8` (текущий рекомендуемый дефолт Anthropic), короткий вывод
(max_tokens 1024). Фича env-gated по паттерну Stripe: без `ANTHROPIC_API_KEY`
эндпоинт отвечает `NOT_IMPLEMENTED` (501), клиент показывает сообщение об ошибке —
сборка и тесты не требуют ключа. Ретраи 429/5xx — встроенные в SDK. Промпт требует
«только текст описания, без выдуманных характеристик»; ответ обрезается до 5000
символов (граница `description` схемы товара).

## Rate limiting — in-memory фиксированное окно (ТЗ §13)

ТЗ предполагает общий счётчик в Memorystore (Redis). Для MVP реализован
**in-memory фиксированное окно** (`middleware/rateLimit.ts`): достаточно для одного
инстанса Cloud Run, без дополнительной инфраструктуры и расходов. Чистое ядро
`fixedWindowHit(store, key, now, windowMs, max)` покрыто unit-тестами; ключ — `uid`
для авторизованных либо IP. Лимиты по чувствительности эндпоинта: чекаут 30/мин,
AI-ассист (платный внешний вызов) 20/мин, продуктовые события 120/мин. Ответ при
превышении — `429 RATE_LIMITED` + заголовки `RateLimit-*`/`Retry-After`.
Ограничение: при горизонтальном масштабировании счётчики не разделяются между
инстансами — выносится в Memorystore тем же контрактом `hit()` (INCR+TTL) перед
включением автоскейла. Отложено до нагрузочного профиля (Фаза 5).

## Итоговая экономическая позиция

Решения минимизируют постоянные и инфраструктурные расходы MVP: один регион вместо
мульти-региона, отказ от второй (веб) платформы, перекладывание платёжного комплаенса
на Stripe, эвристические рекомендации и текстовый AI без дорогой инфраструктуры.
Дорогостоящие опции (мульти-регион, веб-витрина, ML-рекомендации, визуальный поиск)
сохранены как путь развития после подтверждения спроса.
